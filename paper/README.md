# aom-nirs paper

Manuscript and supplement for the Talanta paper *Operator-adaptive PLS and
Ridge calibration for NIR spectroscopy*.

## Build

```bash
bash build.sh
```

Calls `scripts/make_figures.py` (regenerates the matplotlib concept figures
only; the data-driven figures live as pre-built PDFs under `figures/`),
then runs `pdflatex` + `bibtex` + `pdflatex` × 2 for both `main.tex` and
`supplement.tex`. Produces `main.pdf` / `supplement.pdf` plus
`AOM-paper.pdf` / `AOM-supplement.pdf` copies.

Requirements: a TeX Live installation with `pdflatex`, `bibtex`, and the
packages used by `main.tex` (most are in the LaTeX `texlive-latex-recommended`
metapackage; on Ubuntu/Debian: `apt install texlive-full`).

## Review dossier

The `review/` directory contains everything needed to reason about the
paper's claims, audit the data, and migrate the code base if needed:

- `talanta_review.md` — internal review with weakness ranking and fix plan
- `aom_code_inventory.md` — per-file inventory of every AOM source module
- `aom_lib_migration_plan.md` — the three repo options that led to `aom_nirs`
- `pls4all_integration_eval.md` — convergence path with the C++ engine
- `final_stats.md` — current paired-stats numbers (Wilcoxon Holm, Friedman,
  Cliff's delta, runtimes, seed stability)
- `v3_stats.md` — FastAOM-specific stats
- `classification_stats.md` — AOM-PLS-DA and AOM-Ridge classifier stats
- `missing_datasets_per_variant.md` — cohort coverage audit
- `cohort_manifest.csv` — the 61-row regression cohort used by the paper
- `failure_mode_table.csv`, `operator_frequency.csv`,
  `selector_diagnostics.csv`, `selector_seed_stability.csv` — supporting
  evidence files

### Regenerating figures and tables from data

```bash
python paper/review/aggregate_stats.py
python paper/review/build_paper_figures.py
python paper/review/selector_diagnostics.py
```

`aggregate_stats.py` reads result CSVs under `benchmarks/runs/` and writes
LaTeX tables to `paper/tables/` plus pareto/runtime figures to
`paper/figures/`. `build_paper_figures.py` is a companion builder that
covers v3 (FastAOM + cartesian-HPO) evidence. `selector_diagnostics.py`
emits the operator-frequency / failure-mode tables.

After regeneration, rerun `bash build.sh` to refresh `main.pdf`.
