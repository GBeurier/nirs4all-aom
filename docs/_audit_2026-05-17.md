# aom_nirs repository readiness audit (2026-05-17)

Scope: verify that this repository can serve as the public code and result
home for the current arXiv v2 draft.

## Current Status

- Package metadata: `pyproject.toml` exposes `aom-nirs` v0.1.0 and points to
  `https://github.com/GBeurier/aom-nirs`.
- Citation metadata: `CITATION.cff` is present and cites `aom-nirs` as the
  software artifact.
- Examples: `examples/paper_smoke.py` plus focused quickstarts are present.
- Documentation: `docs/architecture.md`, `docs/math.md`,
  `docs/benchmark_protocol.md`, and `docs/reproducibility.md` are present.
- Paper-tied result CSVs are stored under `benchmarks/runs/`.
- The manuscript and supplement are in `paper/`, with the arXiv bundle at
  `paper/aom_arxiv_v2.tar.gz`.

## Paper Consistency Checks

- The public software entry point in the manuscript is now `aom_nirs`.
- `nirs4all` is cited only for NIRS instrumentation, acquisition, and
  provenance context.
- No desktop-application reference is present in the active manuscript sources.
- The main regression denominator is the strict intersection `N_cap = 32`.
- The per-dataset score table has 32 unique datasets and 512 score rows
  (16 variants per dataset).
- The supplementary manifest table is explicitly described as the full
  61-regression + 17-classification manifest, not as the `N_cap = 32` score
  denominator.

## arXiv Packaging

Recommended bundle contents:

- `main.tex`, `main.bbl`
- `supplement.tex`, `supplement.bbl`
- `references.bib`
- referenced `figures/*.pdf`
- referenced `tables/*.tex`

Excluded from the upload bundle: generated PDFs, logs, aux files, PNG previews,
review scripts, caches, local benchmark logs, and previous archives.

## Residual Notes

- `main.tex` and `supplement.tex` are independent LaTeX documents. When
  uploading to arXiv, select the intended top-level document(s) explicitly.
- The generated PDFs compile locally; remaining warnings are non-fatal table
  overfull/underfull boxes caused by long dataset and model labels.
