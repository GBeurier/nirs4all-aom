*"Operator-adaptive PLS and Ridge calibration for NIR spectroscopy"* and that
a fresh clone allows perfect reproducibility of every paper claim.

Audit method: manual walk of every top-level directory + cross-check of paper
data-source paths.

## 2. What is MISSING

### 2.1 BLOCKERS for paper-citation readiness

#### B1 — `examples/` directory is **EMPTY**

The pyproject.toml advertises the package as ready, the README promises `examples/paper_smoke.py`, but the directory is empty. A reviewer who clones the repo cannot run a single end-to-end example.

**Severity:** blocker — every reviewer expects `pip install . && python examples/<something>.py`.

**Required content (minimum):**
- `examples/01_aom_pls_quickstart.py` — fit `AOMPLSRegressor` on a synthetic dataset, print RMSE
- `examples/02_aom_ridge_blender.py` — fit `AOMRidgeBlender` on a synthetic dataset
- `examples/03_fastaom_quickstart.py` — fit `FastAOMPLSRidge(model='sparse_mkr_compact')`
- `examples/paper_smoke.py` — reproduce a 3-dataset smoke of AOM-PLS + AOM-Ridge + FastAOM, runs in < 5 min, prints the paper's headline numbers on that smoke

#### B2 — `docs/` directory is **EMPTY** (except this audit file)

No architecture overview, no math derivations, no benchmark protocol, no reproducibility runbook. The paper's review documents are in `paper/review/` but they are not the kind of "library docs" a user expects.

**Severity:** blocker — paper supplement points to `docs/` and `aom_nirs/docs/`.

**Required content (minimum):**
- `docs/architecture.md` — package layout, class hierarchy, dataflow
- `docs/math.md` — covariance identity, NIPALS derivation, SIMPLS derivation, AOM-Ridge dual formulation, FastAOM screening score
- `docs/benchmark_protocol.md` — how a paper benchmark is run, cohort definitions, seed protocol, stats methodology
- `docs/reproducibility.md` — concrete runbook to reproduce each headline number
- `docs/pls4all_parity.md` — current parity status against `pls4all` C++ ABI

#### B3 — **External paper-tied benchmark outputs** referenced by `paper/review/paper_data.py` / `aggregate_stats.py` live OUTSIDE `aom_nirs/`

Specifically:
- `paper/review/paper_data.py:29-30` → `bench/scenarios/runs/paper_aom_linear_hpo_full_cartesian_default_cv5_all/results.csv` (PLS/Ridge default CV5 baseline)
- `paper/review/paper_data.py:48,56` → `bench/scenarios/runs/paper_aom_linear_hpo_full_cartesian_{pls,ridge}-tabpfn-hpo-*_seed[0-2]/results.csv` (HPO baselines)
- `paper/review/aggregate_stats.py:51` → `bench/scenarios/runs/paper_aom_aompls_seeds012/results.csv` (AOM-PLS regression seeds 0/1/2)
- And FastAOM seed runs at `bench/scenarios/runs/paper_aom_fastaom_*`

These paths are relative to `nirs4all/` (where the original benchmark workspace was). Total size of `results.csv` files is **~7.7 MB**, full directories ~12 MB.

**Severity:** blocker for "perfect reproducibility from a single clone".

**Fix:** copy at least the `results.csv` files (and useful sidecars) from these `bench/scenarios/runs/paper_aom_*/` directories into `aom_nirs/benchmarks/runs/{pls,ridge,fast}/scenarios/`. Then update `paper/review/paper_data.py` and `aggregate_stats.py` to point at the new in-repo locations.

### 2.2 MAJOR for academic / packaging quality

#### M1 — No `CITATION.cff`

Most modern journals expect a machine-readable citation file. The `pls4all` repo has one; `aom_nirs` should too.

#### M2 — No `CHANGELOG.md`

Version 0.1.0 is the first release. A 5-line CHANGELOG documenting the migration is enough but it should exist.

#### M3 — No `MANIFEST.in`

Without it, `pip sdist`/`pip wheel` won't include the paper/tests/examples in the source distribution. Either add `MANIFEST.in` or use the `[tool.setuptools.package-data]` table in pyproject.toml to declare what ships with the wheel.

#### M4 — No README per subdirectory

- `tests/README.md` — how to run tests, what each subdir covers, pytest markers
- `benchmarks/README.md` — how to launch a cohort run, where results go
- `paper/README.md` — how to rebuild the manuscript (mentions build.sh)
- `examples/README.md` — once examples exist

#### M5 — `tests/conftest.py` top-level

The fast/ and ridge/ subdirs have conftest. pls/ doesn't. A shared top-level conftest with `numpy.random` seeding fixtures and a "skip if pls4all parity unavailable" mark would help.

### 2.3 MINOR / nice-to-have

- `paper/main.pdf` may be stale (last build before migration). `paper/build.sh` should regenerate it.
- No CI: `.github/workflows/{ci,release}.yml` deferred to post-submission.
- No `.readthedocs.yml` for hosted docs (deferred).
- `aom_nirs/experimental/` directory is empty; either drop it or add `__init__.py` with a stub.

## 3. Cross-checks

- `paper/main.tex` does NOT reference external `bench/...` paths (good).
- `paper/build.sh` only calls `paper/scripts/make_figures.py` (matplotlib-only) — paper CAN be built from `aom_nirs/` alone with the pre-generated figures.
- All AOM-Ridge classifier / Blender / AutoSelector tests pass (verified previously: 9 pls + 16 ridge + 10 fast tests green from clean checkout).
- `aom_nirs/ridge/__init__.py` correctly does NOT expose `residual_tabpfn` / `tabpfn_candidate` in `__all__` (gated for the optional `[tabpfn]` extra).
- `aom_nirs/pls/preprocessing.py` is self-contained (vendored `ExtendedMSC` + direct `pybaselines` call for `ASLSBaseline`, no `nirs4all` import).
- `aom_nirs/ridge/_spxy.py` is self-contained (vendored `SPXYFold`).

## 4. Summary verdict

The repository **structure** is correct; the **content** has three blockers
(B1-B3) and four major gaps (M1-M5). Once those are fixed the repo is
paper-citation ready and supports perfect reproducibility from a single
clone.

Estimated effort to close:
- B1 + B2 + M4: ~4 dev-hours (writing examples + docs from existing material)
- B3: ~30 minutes (copy 7.7 MB of results.csv + rewrite paths in paper_data.py)
- M1 + M2 + M3 + M5: ~30 minutes (boilerplate files)
- **Total: under one workday.**
