# Graphical abstract

## Concept

The graphical abstract communicates the *operator-as-selector* idea in a
single horizontal flow. From left to right:

1. **Operator bank** ($\mathcal{B}=\{A_1,\ldots,A_B\}$). Identity,
   Savitzky-Golay smoothers and derivatives, finite differences,
   detrend projections, Whittaker smoothers, Norris-Williams gap
   derivatives. Boxes are colour-coded by operator family.
2. **Selection policy**. Five branches stem from the bank: `none`,
   `global` (AOM), `per_component` (POP), `soft`, `superblock`. The
   AOM branch shows one operator selected globally; the POP branch
   shows a different operator at each latent component; the
   `superblock` branch shows the bank concatenated.
3. **PLS core**. The covariance-space SIMPLS engine sits at the
   centre, annotated with the central identity
   $(X A^T)^T Y = A X^T Y$ to underline that operator selection
   happens directly on the cross-covariance.
4. **Tasks**. Two outputs branch on the right: regression
   ($\hat{Y}$) and PLS-DA with class-balanced coding
   ($\hat{p}(c\mid x)$).
5. **Diagnostics**. A small inset shows the per-component operator
   path that POP records, illustrating the auditability of the
   pipeline.

The visual budget should be one row of left-to-right boxes with
horizontal arrows; no nested figures, no equations beyond the central
identity. A subtle teal/cyan colour palette is used for the PLS core
to match the `nirs4all` branding, with neutral greys elsewhere.

## Source figure

The graphical abstract is generated by extracting the
**Operator-Adaptive PLS framework** schematic produced by
`bench/AOM_v0/publication/scripts/make_figures.py`, namely
`figures/fig_framework.pdf`. The graphical abstract is produced by
exporting the schematic at PNG resolution suitable for the journal
(width 2 columns, height 5 cm), which matches the Elsevier graphical
abstract requirements:

- File: `figures/fig_framework.pdf` (vector source)
- Export: `graphical_abstract.png`, 600 dpi, 8 cm height max,
  20 cm width max.

A separate PNG export step (manual or via `pdftocairo -png -r 600`)
should be run by the corresponding author when preparing the journal
submission package.
